## 1. SCQA 执行摘要

**S Situation**  
工贸企业安全监管已经具备明确政策压力和业务抓手：重大事故隐患动态清零、有限空间全过程管理、危险作业审批、精准执法、科技赋能都已成为监管部门和园区的共同议题。

**C Complication**  
但现有方案容易把问题做成“企业台账 + 隐患台账 + 大屏展示”。这种路径能提高填报效率，却无法回答三个主官问题：哪个风险点正在失控、谁应该在多长时间内处理、处理后风险是否真正降级。

**Q Question**  
一期到底应该先建设什么，才能证明数字化监管不是展示工程，而是能减少重大隐患反复出现的治理系统？

**A Answer**  
一期不应先追求全量系统，而应围绕风险点最小治理单元打穿四个闭环：风险点本体建档、危险作业过程留痕、隐患治理证据链、事故险情复盘回写。只要这四个闭环可派单、可降级、可追责，后续 L2-L5 扩展才有真实治理底座。

## 3. 9 章 Pyramid 化报告骨架

| 章节 | Action Title |

| P0 封面 | 工贸安全数字化监管的10倍价值不来自再建一个隐患台账，而来自把风险点本体、危险作业票和复盘回写接成同一个可派单、可降级、可追责的治理闭环。 |

| P7 行动路径 | 0-3 个月只做重点场景闭环，3-6 个月扩展分级监管，6-12 个月再做跨园区复制。 |
| P8 决策清单 | 请先决定一期闭环指标、试点行业、风险点编码口径、派单责任边界和复盘回写规则。 |

## 4. 每章原子单元样例

### P2 风险点本体

**Governing Thought**  
企业只是风险容器，真正需要编码、画像和降级的是有限空间、粉尘涉爆、熔融金属等风险点。

**证据**  
原建议书已列出风险点、设备、物料、作业、责任、时空四类本体要素；高危场景包括粉尘涉爆、有限空间、金属冶炼、液氨制冷、危险作业。

**So-What**  
若仍按企业建台账，监管只能知道“谁有问题”；按风险点建模，监管才能知道“哪个点、因为什么、正在走向什么事故链”。

**Action**  
一期必须输出风险点编码规则、风险点档案模板、重点场景分类表和风险降级条件，不把它们推迟到平台上线后再补。

## 6. 8 反模式扫描

| 反模式 | 命中 | 处理建议 |
|---|---:|---|
| 现状描述癖 | 中 | 背景部分压缩 40%，把政策描述改成“为何现在必须做一期闭环”。 |
| 数据堆砌癖 | 低 | 数字不多，但需补核心基线：企业数、风险点数、隐患闭环率、作业票线上率。 |
| 总分总僵化 | 中 | 当前结构偏项目建议书，应改为 P0-P8 金字塔结构。 |
| 标题主题化 | 高 | “建设内容”“主要功能模块”等标题需改成完整论断。 |
| 假装客观 | 中 | 需要给出推荐优先级：一期先风险点本体和危险作业闭环。 |
| 假设隐性 | 高 | 成本、接入率、AI 识别准确率、监管协同强度都要进入假设表。 |
| 数据约束藏起来 | 中 | 脱敏说明应上移到封面元信息层和附录 C。 |
| 无版本演化 | 高 | 需要 v0.1 -> v0.2 的改写记录和决策变化表。 |
