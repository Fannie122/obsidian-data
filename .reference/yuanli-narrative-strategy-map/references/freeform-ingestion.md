# Free-form 逐字稿摄取协议 · v2 新增

> **何时走这条路**：学员丢进来的是**一堆原始口语逐字稿**（多份会议/访谈/自述录音转写，无 `## Q1`~`## Q20` 结构、顺序乱、有冗余、口语化），而不是干净的 20 题问卷。
>
> **与结构化模式的根本区别**：结构化模式**顺着问题走**（Q1→Q20 逐题对位）。free-form 模式**顺着目标字段走**（拿着 20 个 schema 字段当鱼叉，去整个语料里捞证据）。这是一次**抽取方向的反转**：从 push（问题驱动）变成 pull（字段驱动）。

---

## Phase A · 语料归并（Corpus Intake）

1. **收齐**：用户可一次给多个文件路径或多段文本。全部读入，按来源标号：`[T1 2026-05-12 复盘]` / `[T2 客户访谈]` / `[T3 自述语音]` …
2. **体量闸**：统计全语料净字数。
   - `< 3000 字` → **警示**：free-form 信号被稀释，比干净问卷需要更多原料。建议补录或先走结构化访谈。可继续，但提示成稿会偏空。
   - `≥ 8000 字` → 健康，足以填满 6 节。
3. **去重**：口语转写常有大段重复（同一观点在多次会议反复讲）。合并同义段，**保留最完整/最早的一次原话**作 evidence（不丢信息，只去冗余）。
4. **主体确认（多人语料护栏）**：若语料里出现多个说话人（主持人 + 学员 + 客户），**先锁定哪一位是 subject**。把 subject 的原话与他人原话物理区分；他人原话**只能**作为旁证（如客户证言进 `g2.validation_matrix`），**不能**当 subject 自己的觉醒/母题。
   - 不确定时 → 停下来问用户「这堆逐字稿里，要做战略地图的是谁？」

---

## Phase B · 字段驱动捞取（Field-Driven Harvest）

**不要从头读到尾顺序抽。** 拿着 [extraction-schema.md](extraction-schema.md) 的 ~40 个字段，**逐字段**对全语料做一次定向扫描：

| 字段族 | 在语料里捞什么信号（关键词 / 语气线索） |
|---|---|
| `prologue.borrowing_momentum` 借势 | "现在大家都…"、"这两年变了"、"AI/周期/人群迁移"、对时代趋势的判断 |
| `prologue.combining_forces` 合力 | "我一直擅长…"、"换什么行业我都能…"、跨载体不变的内核能力 |
| `g1.non_rational_obsession` 非理性痴迷 | "做这个我不累"、"赚不赚钱我都…"、"一做就忘了时间"、做了 5-10 年还在做 |
| `g1.asymmetric_advantage` 非对称优势 | "别人学不来"、"我比同行省力"、模仿壁垒 / 认知偏差 |
| `g1.dual_personality` 骑象人×大象 | 理性决策 vs 感性冲动的拉扯、"我知道该…但我忍不住…" |
| `g1.shadow_to_superpower` 阴影→超能力 | 童年/过往痛点 + 它如何反向长出当下能力 |
| `g1.founding_motif` 母题 | 反复出现的那个**动词**（看见/翻译/守护/连接…）—— 跨故事高频复现的那一个 |
| `g2.sweet_user_profile` 甜用户 | 客户画像、谁付钱最爽、谁复购、客单价 |
| `g2.value_pyramid` 价值阶梯 | 客户嘴上买功能、实际买情绪/身份/意义的话 |
| `g2.killer_one_liner` 一句入脑 | 客户**原话复述**过的那句、能被转介绍的那句 |
| `g2.fourth_species` 第四种物种 | "我们既不是 X 也不是 Y"、与 3 类同行的差异 |
| `g3.front/back/finance_chain` 三链路 | 获客渠道 / 交付卡点 / 利润结构、"我自己离不开"、"60 天我走了会不会塌" |
| `g3.kpi_panel` 关键数字 | 任何具体数字（客单 / 转化 / 复购 / 月流水 / 周期）|
| `g4.four_dim_score` 虚实入出 | 品牌溢价 / 规模 / 转换成本 / 网络效应的只言片语 |
| `g4.sixteen_controls` 控制点 | 护城河相关的零散表述，缺的填占位评分（标 `inferred`）|
| `closing.vision_paint` 远方 | "我最想…"、"十年后…"、画面感的愿景段落 |

**捞取规则**：
- 一个字段可能散落在多份逐字稿的不同位置 → 全部扫到，取最有力的一条当主 evidence，其余可并入。
- evidence 必须是**逐字原话 30-80 字**，标注来源 `[T2]`。**禁止转述、禁止润色后再当原话**。

## Phase C · 证据绑定 + 置信度（free-form 强制项）

结构化模式里字段要么有要么 `null`。free-form 里多了一个**置信度**维度，必须显式报：

```json
"g1.founding_motif": {
  "verb": "看见",
  "sentence": "...",
  "evidence": "[T1] 「其实我做的每件事，本质上都是帮一个人被看见」",
  "confidence": "H",        // H 直接明说 / M 多处旁证拼出 / L 弱推断
  "inferred": false          // true = 语料没明说，是 LLM 综合推断
}
```

- `confidence: H` —— subject 原话直接讲清。
- `confidence: M` —— 语料里多处碎片拼出，方向明确但没有一句话总结。
- `confidence: L` —— 弱信号，仅一处暗示。**L 字段在渲染时标灰/降级，不进 HERO 等强位**。
- `inferred: true` —— 语料**没有**直接支撑，是 LLM 从相邻信息推断。**inferred 字段必须在成稿对应模块加脚注**「⊙ 推断项，待学员确认」，绝不冒充原话。

---

## Phase D · 缺口账本（Gap Ledger）

free-form 几乎一定填不满 20 字段。收尾**必须产出缺口账本**，不要假装齐全：

```
## 缺口账本（本次 free-form 摄取）
- 已捞到 H/M 置信字段：14 / 20
- 空字段（渲染砍块）：g2.fourth_species, g4.sixteen_controls(部分)
- 仅 L 置信（降级渲染）：g1.dual_personality
- 二次访谈待补清单：
  - 缺 g2.fourth_species → 问 Q11「3 类典型同行 + 你 = 哪两种混合 + 哪个独有」
  - 缺 g4 评分 → 问 Q16「虚/实/入/出 四维现在各几分」
```

缺口账本同时是**反编造护栏**：把"没问到"显式记下来，模型就不会去用训练先验填它。

---

## Free-form 八条反编造铁律（比结构化更严，因为诱惑更大）

1. 语料没有的，**留 null**，进缺口账本。绝不用"伟大公司都需要…"之类通用话术补。
2. 任何 evidence 必须是逐字原话 + 来源标号，禁止把自己的总结当原话。
3. 字段不串味：g1 的料不许搬去填 g3。
4. 多人语料里，他人原话不许冒充 subject 自己的觉醒/母题。
5. 推断项一律 `inferred:true` + 成稿脚注，绝不静默升格为事实。
6. 置信度必须真实报，不许把 L 谎报成 H 去撑满版面。
7. 数字只能用语料里出现过的；没有就空着，绝不编 KPI。
8. 宁可成稿偏空、模块少，也不要虚高满版 —— **半真的奢侈感 HTML 比诚实的留白更危险**。

## 与结构化模式的接口

- 摄取层产出的 JSON **与结构化模式完全同 schema**（[extraction-schema.md](extraction-schema.md)），只多了 `confidence` / `inferred` 两个字段。
- 因此 Step 3 Render 完全复用，无需为 free-form 单独写渲染逻辑。
- SKILL.md Step 1 的**双模式判定**：检测到 `## Q` 标记 ≥ 10 处 → 走结构化快路；否则 → 走本协议。
